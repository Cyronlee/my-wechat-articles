**使用 https://github.com/doocs/md 编写微信公众号文章时，排版优化可显著提升读者阅读体验。** 该工具的核心优势在于将标准 Markdown 语法实时渲染为微信兼容的简洁、优雅图文样式，避免了传统公众号后台的繁琐调整。以下建议基于 doocs/md 的官方特性（支持 GFM 扩展、自定义主题/CSS、代码高亮、AI 辅助等）、用户社区讨论以及微信公众号阅读场景的通用最佳实践，分为“推荐格式”和“不推荐做法”两部分，便于参考。

### 1. 推荐的格式和排版方式

这些做法能增强文章的层次感、可读性和专业性，充分利用 doocs/md 的渲染能力：

- **标题层级清晰且适度**：优先使用 `## 二级标题` 和 `### 三级标题` 构建结构，避免过多一级标题（`#`），尤其当公众号已设置独立标题时。建议每 300-500 字出现 1-2 个小标题，帮助读者快速扫描。
- **短段落与良好间距**：每段控制在 3-5 行（移动端屏幕约 1/3），段落间自然换行。doocs/md 默认样式已优化行距，建议通过自定义 CSS 微调为 1.6-1.8 倍（社区讨论 #426 常用值），提升呼吸感。

- **列表与强调元素**：大量使用无序列表（`-`）或有序列表（`1.`）呈现要点、步骤或对比，提高可读性。关键句使用 `**粗体**` 或 `*斜体*` 适度突出，切勿过度。支持 GFM 提示框（如 `> [!NOTE]`、`> [!TIP]`、`> [!WARNING]`），可用于知识点、建议或警示，丰富视觉层次。

- **代码块与技术内容**：技术文章强烈推荐使用代码块，并切换至适合的 20+ 高亮主题（支持深色/浅色模式）。Mermaid 图表、PlantUML 和数学公式渲染效果优秀，建议保持代码块简短，每块不超过 15 行。

- **图片与富媒体**：使用一致尺寸的图片（建议宽度 800-1200px），以获得最佳显示效果。支持幻灯片语法 `![alt](url),![alt](url)` 创建横屏滑动图集。插入图片后可通过 Markdown 添加简短说明，提升图文融合感。多图床上传功能可确保加载速度。

- **自定义主题与 CSS 优化**：在编辑器中切换内置主题或使用色盘/自定义 CSS（讨论区提供大量用户分享，如简约技术风、Claude 风格）。推荐调优字体、颜色（不超过 3 种主色）、行高和边距，实现“赛博禅心”或优雅技术风，提升品牌一致性。

- **外部链接处理**：利用编辑器左上角“格式 → 微信外链接转底部引用”功能，将非公众号链接转为底部引用列表，避免微信限制导致的点击失效问题。

- **AI 辅助一键排版**：集成 DeepSeek 等 AI 助手，可快速优化格式、生成提示框或调整结构，进一步提升效率。

上述推荐均可在 doocs/md 预览区实时验证，建议使用 Chrome 浏览器并开启移动端预览，确保最终效果符合微信阅读习惯。

### 2. 不推荐的格式和排版方式

以下做法易导致阅读疲劳、排版混乱或渲染不佳，应尽量避免：

- **长段落或密集文本**：避免单段超过 6-8 行或“大段文字墙”，这在移动端会显著降低阅读意愿。doocs/md 虽能渲染，但长段落会削弱其简洁优势。

- **过度复杂或不一致的样式**：勿频繁切换字体颜色、背景或使用过多 Markdown 扩展（超出 GFM 支持范围），以免在微信后台粘贴后出现错乱。自定义 CSS 也应保持简洁，避免复杂选择器导致加载变慢或样式冲突。

- **一级标题滥用与层级混乱**：公众号文章已有独立标题栏，文章内过多 `#` 会显得冗余且破坏 SEO 结构。标题层级必须严格递进（H2 > H3），不可跳级。

- **图片尺寸不统一或过多**：大小差异过大易造成布局跳跃；连续插入过多图片而不加说明会干扰阅读节奏。避免使用低分辨率或横竖混杂的图片。

- **首行缩进或传统排版习惯**：微信阅读场景下，首行缩进反而降低现代感，doocs/md 默认无缩进样式更佳。
